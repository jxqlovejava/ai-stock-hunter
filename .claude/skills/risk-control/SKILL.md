# 风控执行 (Risk Control)

硬性风控约束，不可被任何上游信号覆盖。

## 硬约束规则

| 约束 | 阈值 | 逻辑 |
|------|------|------|
| 单票上限 | 20% | 单一股票不超过总仓位 20% |
| 行业上限 | 40% | 单一行业 (申万一级) 不超过 40% |
| 单笔止损 | -2% | 单笔交易亏损超过 2% 强制平仓 |
| 组合回撤熔断 | -15% | 组合总回撤超过 15% 禁止新开仓 |
| 黑天鹅熔断 | 沪深300 单日 -5% | T+1 禁止新开仓 |
| 双创折扣 | ×0.8 | 科创板/创业板波动高，仓位打折 |
| 流动性上限 | 持仓<5%日均成交额 | 避免流动性枯竭 |
| 系统熔断 | 滚动 3 月胜率<40% | 策略失效，暂停交易 |

## 输出

```python
RiskCheck(
    passed=True,               # 是否通过所有硬约束
    violations=[],             # 违规记录
    adjusted_weight=0.12,      # 调整后仓位 (被上限裁剪后)
    source_citations=[...],    # 继承引用链
)
```

## 工作流

1. 接收 `PositionSignal` + `portfolio` + `market` 上下文
2. 逐条检查硬约束
3. 按优先级裁剪仓位 (回撤熔断 > 黑天鹅 > 单票上限 > 行业上限 > 双创折扣)
4. 返回 `RiskCheck`

## 护栏

- **不可覆盖**: 风控约束 = 硬性，无例外
- **先裁后报**: 先裁剪仓位，再报告违规
- **全或无**: 回撤/黑天鹅/系统熔断触发时新仓为 0
