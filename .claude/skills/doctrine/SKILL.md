# 军规系统 (Doctrine)

31 条交易军规，8 大类，3 级严重度。所有交易决策前强制执行。

## 分类概览

| 类别 | 规则数 | 示例 |
|------|--------|------|
| 仓位管理 | r001-r004 | 永不满仓、单票≤20%、分批建仓 |
| 选股 | r005-r008 | 拒绝 ST、不在财报前赌博、不追涨杀跌 |
| 交易执行 | r009-r015 | 不追涨停、不接飞刀、止损后休息 |
| 情绪控制 | r016-r019 | 恐慌不割肉、连赢不膨胀 |
| 信息管理 | r020-r024 | 零信任传闻、交叉验证、来源分级 |
| 风控纪律 | r025-r028 | -2% 止损、-15% 熔断、不补亏损仓 |
| 复盘学习 | r029-r030 | 每笔交易记录理由、每周复盘 |
| 元规则 | r031 | 系统胜率<40% 暂停交易 |

## 严重度

| 级别 | 含义 | 动作 |
|------|------|------|
| BLOCK | 致命违规 | 阻止交易，不可覆盖 |
| WARN | 警告 | 降低信心度，需人工确认 |
| INFO | 提示 | 记录但不禁行 |

## 工作流

1. 接收股票代码 + 上下文 dict
2. 逐条评估规则
3. 收集 BLOCK/WARN/INFO
4. 返回 `DoctrineResult`

## 输出

```python
DoctrineResult(
    passed=True/False,
    blocked_by=[Rule, ...],  # BLOCK 级别的规则
    warnings=[Rule, ...],    # WARN 级别的规则
    infos=[Rule, ...],       # INFO 级别的规则
)
```

## A 股特定

- r006: ST/*ST 股票一律排除
- r015: 财报公布前 3 日禁止开新仓（A 股业绩变脸风险）
- r022: 财务数据必须至少 2 个独立来源交叉验证
- r024: 微信/QQ/论坛传闻零信任

## 护栏

- **先于所有分析**: 军规在 L0 之前执行
- **BLOCK 不可覆盖**: 无例外
- **可审计**: 每条规则可追溯到编号
