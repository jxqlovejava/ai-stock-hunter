## 前言：毒辣拷打 10 问与回应

> 以最挑剔的投资者的视角，对 v6 计划进行了系统性拷打。以下是发现的致命问题及 v7 回应。

### 🔴 拷打 1：这个系统到底能不能赚钱？证据在哪？

**问题**：30 条军规 + 5 层路由 + 232 个模型 + 14 个情绪信号——这是一座精美的大教堂。但没有任何证据证明其中任何一个组件能产生超额收益。金融工程的第一条铁律是：**先证明简单的东西能赚钱，再往上加复杂度。** 你现在是反过来，先把复杂度堆满，然后指望回测来验证。这是过拟合的完美配方。

**v7 回应**：Phase 1 回测必须是**最小可行策略**，不是「宏观打分→沪深300 ETF」这种简单到没有信息量的基准。最小可行策略 = 2-3 个因子 + 1 条风控规则，先跑出正期望值。军规/Munger/情绪套利在回测中作为**可选叠加层**，每加一层必须证明边际贡献为正。**不能证明边际贡献的组件，不进实盘。**

### 🔴 拷打 2：数据基础设施都没搭好，5 层路由是空中楼阁

**问题**：华泰、国信、VSignals、AKShare 四个数据源，你能保证同一只股票的 PE 在这四个源上是一致的吗？如果华泰说 PE=15.2，国信说 PE=16.8，AKShare 说 PE=14.5——哪个是真相？L1 分析的量化因子如果吃的是互相矛盾的数据，L2 法官的评分就是垃圾。

**v7 回应**：事实性校验管道优先级提升到 Phase 1.7（原 1.6）。新增规则：**同一数据点 ≥ 2 源交叉验证，差异 > 5% 时标记 DISPUTED，不进入分析管道。** 在 Phase 1 必须完成 A 股全市场 PE/PB/ROE 数据一致性审计——搞清楚不同数据源的差异有多大，哪些数据根本不可信。

### 🔴 拷打 3：小白用户不需要「三色信号」，需要的是「阻止自毁」

**问题**：计划说小白看到 ✅/⚠️/🔴 三色信号。但真正的问题是——小白会听吗？小白看到 ⚠️ 说「不建议追高」，然后看着股价继续涨，心里想「这破系统不行」，手动追进去。系统的输出如果用户不执行，等于不存在。对于小白，**阻止一次冲动交易的价值 > 推荐 10 只好股票的价值**。

**v7 回应**：军规层的硬阻断优先级高于一切分析。r012（不追涨停）、r015（不赌财报）、r018（恐慌不决策）必须在 CLI 层面物理阻断操作路径——不是「建议不要买」，而是「系统拒绝执行此操作，T+1 冷静期后再来」。配合防线 3 的「事后对比」——用户手动操作被事后打脸——逐步建立信任。

### 🔴 拷打 4：恐慌套利引擎需要 Bloomberg 级基础设施

**问题**：Meta 宣布输出算力 → 5 分钟内识别受影响的 A 股存储标的 → 估算 EPS 冲击 → 计算合理跌幅 → 判定过度反应。这个链路的每一步都需要：实时新闻流（秒级延迟）、NLP 实体识别与股票映射、基本面模型。AKShare 的新闻模块做不到这些。你用 AKShare 做恐慌套利，等于用自行车跑 F1。

**v7 回应**：恐慌套利分两阶段实现。**简版（Phase 2）**：只追踪可量化的大盘/板块情绪极端值（涨跌比、成交额异常、北向资金突变），不依赖实时新闻 NLP。这些数据 AKShare 可以覆盖。**完整版（Phase 4+）**：等华泰/国信 skill 的实时新闻推送能力验证通过后，再上 NLP+事件驱动。**简版能捕捉的情绪套利机会虽然比完整版少，但信号质量更高、幻觉更少。**

### 🔴 拷打 5：10 个宏观指标都是美国的，A 股靠美国宏观来择时？

**问题**：ISM PMI、TIPS 盈亏平衡率、CME FedWatch、DXY、EUR/USD… 这些全是美国宏观指标。你用它来对 A 股择时？中美经济周期脱钩已经是共识了。2024-2025 年中国经济疲软但 A 股反弹，美国经济强劲但 A 股不跟——你的宏观打分卡在脱钩环境下会系统性误判。

**v7 回应**：宏观指标拆分为两组。**全球组（原 10 指标中保留 5 个）**：铜金比、DXY、CNH/USD（这三个直接传导到 A 股资金面）+ 高收益利差、FedWatch（全球风险偏好）。**A 股组（新增 5 个）**：中国制造业 PMI、社会融资规模增速、LPR 利率、沪深300 风险溢价（ERP = 1/PE - 10Y国债收益率）、A 股成交额变化趋势。**全球组和 A 股组分别打分，加权合并。A 股组的权重 > 全球组。**

### 🔴 拷打 6：30 条军规同时生效时，你几乎没法交易

**问题**：r001-r005 仓位限制 + r012 不追涨停 + r013 不接飞刀 + r015 不赌财报（每年 4 次 × 持仓股票数）+ r018 恐慌不决策（每年约 20 个恐慌日）+ r019 盈利上移止损 + r022 信源交叉验证……这些规则叠加的结果是：**你的交易窗口极小，大部分时间系统都在阻止你交易。** 如果这就是目的——很好。但如果是这样，L1 分析、因子研究、情绪信号的投入产出比就值得怀疑——你花 80% 的工程时间建的分析引擎，只在 20% 的时间里产出可执行的信号。

**v7 回应**：承认这个矛盾是系统设计的核心张力。解决方案：**分层启用。** 小白模式：15 条军规全部强制（阻止自毁优先于捕捉机会）。进阶模式：block 级保留，warn 级可配置。专业模式：仅保留 r001/r002/r025/r026 四条最核心的 block 级规则。**系统的默认立场是保守，用户随着能力提升逐步解锁自由度。** 这与用户成长路径（依赖→独立）一致。

### 🔴 拷打 7：Munger 模型「动态匹配」说得天花乱坠，实现起来是玄学

**问题**：「根据上下文从 232 个模型中匹配 2-3 个」——怎么匹配？关键词搜索？Embedding 相似度？还是让 LLM 自己选？如果是关键词，一个包含「PE 过高」的分析会匹配到「安全边际」，但也会匹配到「锚定偏差」「过度乐观倾向」「市场先生」……你能保证每次匹配的都是最相关的吗？如果匹配错了，这个功能就是噪声生成器。

**v7 回应**：匹配机制明确化。**第一步**：基于规则的硬匹配——标的特征（行业/市值/估值分位）+ 市场状态（恐慌/贪婪/中性）+ 分析阶段（选股/估值/择时/风控）→ 每个维度预定义映射表，从 232 缩到 15-20 候选。**第二步**：LLM 从候选中精选 2-3 个最相关的，输出匹配理由。**匹配理由本身可被质疑和修正**——如果用户说「这个模型不适用」，系统记录并调整映射表。**底线：宁可不匹配，不可乱匹配。** 如果候选中没有置信度 > 0.7 的匹配，标注 `[无可匹配模型]`，不强制输出。

### 🔴 拷打 8：系统没有「系统级熔断」——如果整个模型连续错 6 个月怎么办？

**问题**：L4 风控有单笔止损、组合回撤熔断，但这些都是**单笔/组合层面**的。如果系统整体的宏观判断连续错 6 个月——比如一直看多但市场一直跌——系统凭什么继续给出交易建议？你需要一个**元层面的熔断**：当系统整体胜率跌破某个阈值时，全局静默，只保留 L4 止损功能。

**v7 回应**：新增 r031：**系统级熔断**。

| ID | 类别 | 名称 | 严重度 | 核心内容 |
|----|------|------|--------|---------|
| r031 | 元风控 | 系统级熔断 | **block** | 系统整体建议的滚动 3 月胜率 < 40%（或显著低于基准），触发全局静默模式：暂停所有开仓建议，仅保留持仓的 L4 止损监控，强制人工复核后才能恢复 |

同时新增月度系统健康度报告：胜率趋势 / 夏普趋势 / 最大回撤 / 信号产出量 / 各因子 IC 趋势。**绿灯（正常）/ 黄灯（警惕）/ 红灯（熔断）。**

### 🔴 拷打 9：用户成长路径和系统复杂度是矛盾的

**问题**：计划说「用户应该越来越不需要系统」，但系统有 30 条军规、232 个模型、5 层路由、14 个情绪信号……这个复杂度本身就是依赖性的来源。真正好的投资教练，教的是 3-5 条核心原则，而不是一套你永远无法内化的复杂系统。**你在用造航天飞机的方式教人骑自行车。**

**v7 回应**：承认矛盾，但不回避。解决方案：**每一层输出都附带「如果你要自己判断，只看这 3 个指标」的精简版。** 比如 L1 分析完整输出是 20 个指标 + AI 报告，但精简版是「PE 分位 + ROE 趋势 + 北向资金方向」。随着用户升级（小白→进阶→专业），精简版不变，但用户可以打开更多层。**核心原则只有 5 条**（安全边际、能力圈、市场先生、概率思维、逆向思维），这 5 条是用户最终需要内化的。军规 30 条和模型 232 个是系统的内部框架，不是用户的学习大纲。

### 🔴 拷打 10：计划里没有一个「什么时候弃用这个系统」的答案

**问题**：最好的工具是让你有一天不再需要它的工具。如果用户用了 2 年还是每天打开系统问「能不能买」，说明系统作为教练失败了。你怎么衡量这个系统是否完成了它的使命？

**v7 回应**：在 Phase 4 用户能力画像中增加「独立决策准确率」指标——用户做出的、未受系统建议影响的独立决策，事后胜率如何。如果用户的独立决策准确率连续 6 个月 > 55%（超过散户平均），系统推送祝贺消息：「你已经可以独立投资了。建议将系统从『决策助手』模式切换到『风控安全带』模式。」**系统在设计之初就设定了自己的退出策略。**

## Summary

面向 A 股（主板/创业板/科创板）的 AI 驱动投资决策系统。v6 关键升级：**军规重写为 A 股专属（非黄金规则照搬）、Munger 模型从硬编码 23 个改为 232 个动态模型库匹配**。

核心链路：**投资者画像加载 → 军规门禁 → L0 保安 → L1 分析师（含情绪套利）→ L2 法官 → L3 交易员（仓位计算）→ L4 风控官 → 画像匹配校验 → 输出**。

**本系统的独特定位**：不仅是一个分析工具，更是一个**投资教练**。通过个性化画像约束、决策追踪与事后复盘、系统自我迭代，让用户在使用过程中**自身投资能力持续提升**。

### 持仓拆分：核心仓 vs 交易仓

> 借鉴 ai-gold-miner 的 core/tactical split 设计，适配 A 股场景。

| 仓位 | 占比 | 目的 | 持仓周期 | 止损规则 |
|------|------|------|---------|---------|
| **核心仓** | 60-70% | 长线价值投资，搏取长期复利 | 年级 | 仅基本面逆转或硬止损触发才卖 |
| **交易仓** | 30-40% | 波段/情绪套利，增厚收益 | 周-月级 | ATR 移动止盈 + 单笔 2% 硬止损 |

**不同仓位的操作权限：**
- 核心仓：只能因基本面变化或 L2 评分 < 35 卖出
- 交易仓：L3 信号可灵活操作，恐慌套利专门用交易仓执行
- 交易仓盈利 > 20% 时，可转 50% 到核心仓（利润锁定+底仓积累）

### 仓位计算公式（v5 更新）

```python
# L3 交易员的最终仓位计算，每次决策前先加载 portfolio.yaml

def calculate_position(
    verdict: Verdict,
    portfolio: Portfolio,
    macro_cap: float,          # 宏观打分卡仓位上限
    is_core: bool = False,     # 是否核心仓操作
) -> float:
    # 1. 基础仓位
    base = (verdict.score - 50) / 50 * macro_cap

    # 2. 画像约束
    base = min(base, portfolio.limits.max_single_pct)
    base = min(base, 1.0 - portfolio.limits.min_cash_pct)

    # 3. 交易仓额外折扣（恐慌套利）
    if not is_core and verdict.sentiment_signal == "PANIC":
        base *= 0.5  # 恐慌套利仓位减半

    # 4. 创业板/科创板折扣
    if portfolio.get_stock_market(verdict.symbol) in ("创业板", "科创板"):
        base *= 0.8

    # 5. L4 风控最终检查
    return apply_risk_constraints(base, portfolio)
```

### 画像匹配校验（每次输出必须包含）

> 借鉴 ai-gold-miner 的强制画像匹配步骤。

每次交易建议输出时必须明确回答：
1. **仓位上限检查**：建议仓位是否在单票/单行业/总仓位限制内？
2. **持仓周期匹配**：这笔操作是核心仓逻辑还是交易仓逻辑？
3. **风险偏好匹配**：建议的激进程度是否符合用户的平衡/保守/进取定位？
4. **能力圈检查**：标的所在行业是否在用户熟悉范围内？若不在，标注 `[超出能力圈]`
5. **硬止损触发检查**：当前浮亏是否已触发止损线？

## 投资军规体系（v6 重写：A 股专属）

> v6：黄金投资军规与股票投资有本质差异。黄金是单一品种的宏观对冲工具，股票涉及个股选择、行业轮动、管理层判断。
> 以下 30 条为 A 股场景重新设计，分为 7 大类。严重度：**block**（阻断交易）/ **warn**（警告）/ **info**（信息）。

### 军规总览

### 0.2 小白如何获得超额收益？（四道防线）

小白亏损的根因不是「不会选股」，是**行为偏差**——追涨杀跌、不止损、仓位失控。系统从四个层面构建防线：

**防线 1：阻止自毁行为（最重要的一道）**

| 小白典型错误 | 系统拦截机制 | 实际效果 |
|-------------|-------------|---------|
| 涨停板排队追买 | L0 保安：涨停板直接 REJECT | 物理阻止追涨 |
| 跌停板恐慌出逃 | L0 保安：跌停板不允许卖出指令（已跌停出不去，第二天再判断） | 防止情绪化割肉 |
| 全仓押一只股票 | L4 风控：单票 20% 硬上限，超配无法下单 | 强制分散 |
| 亏损死扛 | L4 风控：单笔亏损 2% 自动触发止损 | 机械止损，无情绪 |
| 听消息买垃圾股 | L0 保安：ST/\*ST、违规记录直接拦截 | 过滤掉最危险的标的 |
| 频繁交易 | 每周操作复盘标注换手率，超阈值提醒 | 意识到交易成本 |

**防线 2：把复杂分析翻译成人话**

```
系统内部输出:                     给小白看:
──────────────────────────────────────────────────
P/E TTM: 85.3 (近5年92%分位)  →  ⚠️ 估值很贵！比过去5年里92%的时间都贵
ROE: 8.2% (连续3年下降)       →  ⚠️ 赚钱能力在变弱，连续3年下滑
MACD 死叉 + 放量下跌            →  🔴 技术面出现卖出信号，有人在大量抛售
L2 置信度: 0.45                →  ⚠️ 系统对这只票的判断不太有把握，不建议操作
```

**防线 3：让用户看到「如果听系统的会怎样」**

- 系统每天生成一份简单的「如果昨天听系统的，今天会怎样」对比
- 用户自己做的交易 vs 系统建议的交易，事后对比盈亏
- 不是羞辱用户，而是建立信任：「你看，按规则来确实比你情绪化操作强」

**防线 4：L4 风控是所有人共用的一道墙**

- 不管你是小白还是专业，L4 的硬约束对所有人都生效
- 专业的可以调高仓位上限（在 L4 允许范围内），但不能关闭止损
- 这是「安全带」——你可以开快车，但不能不系安全带

### 0.3 凭什么有信心？—— 五层信心体系

用户凭什么相信系统不是在胡说八道？需要五层可验证证据：

| 层级 | 证据 | 用户能看到什么 | 更新频率 |
|------|------|--------------|---------|
| **1. 回测证据** | 策略在 2019-2024（含 2018/2022 熊市）的绩效 | 夏普比率、最大回撤、年度收益、胜率 | 策略变更时 |
| **2. 样本外验证** | 最近 6 个月的实盘模拟（非回测） | 模拟组合 vs 基准指数走势图 | 每日 |
| **3. 置信度校准** | 「当系统说 confidence 0.8 时，历史上对的比例」 | 校准曲线：说 80% 就该对 80% | 每月 |

### 0.4 系统自我迭代 —— 五个反馈环

系统不是静态的。以下是五个内置迭代环：

```
环 1: 因子权重自适应（自动）
  每月回测 → IC 分析 → 因子权重微调
  例: 动量因子连续 3 个月 IC 下降 → 自动降低其在 L2 评分中的权重

环 2: 规则阈值自适应（自动）
  风控参数根据市场波动率动态调整
  例: VIX/波动率上升 → 自动收紧单票仓位上限 20%→15%

环 3: 策略有效性监控（自动 + 人工确认）
  策略夏普连续 6 个月 < 0 → 标记为「失效」，降权或下线
  新策略上线 → 3 个月影子运行（不实际执行）→ 验证后切换

环 4: 用户反馈驱动的规则修正（人工触发）
  用户标记「这个分析有问题」→ 记录 case → 回测验证 → 修正或解释
  用户决策数据 → 发现系统性行为偏差 → 增加针对性防护规则
  例: 大量用户在财报前追涨 → 新增规则「财报前 3 天不推荐开新仓」

环 5: 市场结构变迁检测（自动告警 + 人工分析）
  关键因子相关性突变 → 可能意味着市场逻辑变了
  例: 「铜金比与沪深 300 相关性从 0.6 降到 0.1」→ 重新审视宏观指标权重
```

**底线原则：环 1/2/3 的自动调整范围有硬上限。** 比如因子权重单月变化不超过 ±15%，防止系统在回测中过拟合、自我欺骗。

### 0.5 用户成长路径：从依赖到独立

**这是系统最重要的设计目标之一：用户应该在使用过程中变得越来越不需要系统。**

```
阶段 1: 依赖期（0-3 个月）
  用户行为: 完全依赖系统建议，一键跟单
  系统角色: 安全网 + 翻译官
  成长目标: 学会不亏大钱、理解基本概念（PE/ROE/止损）
  
阶段 2: 理解期（3-6 个月）
  用户行为: 看系统分析逻辑，开始有自己的判断
  系统角色: 导师 + 校验器
  成长目标: 能独立判断一只股票的优劣、理解仓位管理
  
阶段 3: 协作期（6-12 个月）
  用户行为: 自己形成选股逻辑，让系统做反向验证
  系统角色: 对手方 + 盲点扫描仪
  成长目标: 有自己的投资体系、能识别系统可能犯错的情况
  
阶段 4: 独立期（12 个月+）
  用户行为: 系统只是工具箱之一，核心决策靠自己
  系统角色: 效率工具 + 风控安全带
  成长目标: 稳定盈利、不再需要「能不能买」这种问题
```

**促进成长的关键设计：**

- **渐进式信息披露**：小白看到 3 个指标 → 进阶看到 15 个 → 专业看到全量原始数据
- **复盘即教学**：每周末推送「本周操作复盘」，不是打分，是问问题——「你这周在XX上赚了钱，是因为你的分析对了，还是运气好？」
- **概念渗透**：系统输出里的每个专业术语都可点击查看白话解释
- **决策笔记**：每次交易建议附带一个必填字段——「你同意/不同意这个建议的理由是？」事后回看
- **能力雷达图**：每月生成用户能力画像——选股能力/择时能力/风控纪律/情绪控制，对比历史趋势

**核心设计原则（v7 更新）：**
1. **先证明简单能赚钱，再加复杂度** — 每加一层必须证明边际贡献为正，否则不进实盘
2. **画像先行** — 在一切分析之前加载用户画像，所有建议必须在用户约束范围内
3. **阻止自毁 > 捕捉机会** — 帮小白避免一次大亏的价值远超帮他找到一只牛股
4. **军规分层启用** — 小白全开(15条)，进阶开 block+warn，专业仅保留 4 条核心 block
5. **系统也要被熔断** — r031：系统整体连续犯错时全局静默，模型不能无限犯错
6. **数据上行，决策下行** — 数据只往上流，指令只往下传
7. **规则引擎做主路径，AI 做分析增强** — 军规/L0/L4 纯规则，L1/L2 量化为主+AI 为辅

### 5.0.0 投资军规 — 硬纪律层（在 L0 之前运行）

军规不是建议，是**不可覆盖的硬纪律**。L4 风控管的是仓位和止损，军监管的是「你该不该做这笔交易」。

```
军规层 → L0 保安 → L1 分析师 → L2 法官 → L3 交易员 → L4 风控官
  ↑
  在一切分析之前先问：这笔交易配不配被分析？
```

| # | 军规 | 触发条件 | 动作 | 不可覆盖? |
|----|------|---------|------|----------|
| 1 | **不懂不投** | 股票所在行业超出用户「能力圈」白名单 | ⛔ 拦截，输出「这只票不在你的能力圈内」 | 是（小白模式）/ 可配置（专业模式） |
| 2 | **永不追高** | 当日涨幅 > 7% 且用户意图为买入 | ⛔ 拦截，「T+1 冷静期后再看」 | 是 |
| 3 | **永不接飞刀** | 连续 3 日跌幅 > 15% 且无基本面变化信号 | ⛔ 拦截，「等止跌确认后再考虑」 | 是 |
| 4 | **利好出尽是利空** | 重大利好公告 + 股价已在 5 日内涨超 15% | ⚠️ 强制标注「利好已定价风险」，置信度自动 -0.15 | 否（标注不可跳过） |
| 5 | **不在恐慌中做决定** | 用户情绪检测为「恐慌」状态（快速连续查询 + 大盘暴跌） | ⛔ 拦截所有买入操作，建议冷静 2 小时 | 是 |
| 6 | **不要爱上你的持仓** | 用户对同一只票连续 3 次拒绝卖出建议 | ⚠️ 强制推送「确认偏误风险」报告 | 否 |
| 7 | **仓位决定心态** | 用户总仓位 > 宏观建议仓位 × 1.5 | ⛔ 拦截新开仓，强制减仓至安全线 | 是 |
| 8 | **亏损是交易的一部分** | 连续止损 3 次 | ⛔ 强制暂停交易 3 个交易日 | 是 |
| 9 | **不要对抗趋势** | 个股处于明确下降趋势（60日均线下方）且宏观评分 < 40 | ⛔ 拦截买入，「不要在熊市里找牛股」 | 是 |
| 10 | **不懂的衍生品不碰** | 用户意图交易可转债/期权等 | ⚠️ 强制弹出风险教育 + 需要二次确认 | 是（小白模式） |

**军规的设计哲学：**
- 前 5 条是格雷厄姆/巴菲特的核心理念，管「什么时候不该动」
- 后 5 条是行为金融学的实操应用，管「你的大脑在骗你」
- 军规的输出不是建议，是**阻断**——在触发条件下，系统直接拒绝执行

### 5.0.1 投资思维模型 — L1 分析师的视角框架

思维模型不直接产生交易信号，而是**约束 L1 分析师看问题的角度**。每个标的在进入因子分析前，必须先过这 5 个思维模型检查：

| 思维模型 | 核心问题 | 应用到分析中 |
|---------|---------|------------|
| **安全边际** | 当前价格比内在价值低多少？ | L2 评分中：安全边际不足（<15%）的标的，基本面得分打折 |
| **护城河** | 公司凭什么挡住竞争对手？ | L1 分析维度：品牌/技术/网络效应/规模/切换成本，5 维度打分 |
| **市场先生** | 当前价格是市场理性定价还是情绪定价？ | **→ 直接连接恐慌套利模块**：情绪定价 = 套利机会 |
| **概率思维** | 这不是「对或错」，是「60% 概率赚 20%，40% 概率亏 5%」 | L2 输出必须有期望值计算，禁止「一定涨」「肯定会跌」 |
| **二阶效应** | 这个事件的一阶影响已经被定价了，二阶影响呢？ | L1 分析模板：事件→直接影响(一阶)→连锁反应(二阶)→谁受益/谁受损 |

**思维模型的落地方式：**
- 不是独立的代码模块，而是嵌入 L1 分析的 **prompt 模板约束**
- 每个模型的检查结果写入 `AnalysisReport.mental_model_checks`
- 任何一个模型未通过，L2 置信度自动打折

## 六、市场情绪与恐慌套利 — 短期套利的核心引擎

> v4 新增。这是 Alpha 三来源之一「适度短期套利」的关键实现。
> 核心理念：市场经常对新闻**过度反应**，情绪驱动的价格波动会**均值回归**。
> 系统要做的不是预测新闻，而是在新闻引发恐慌/狂热时，识别「市场是不是反应过度了」。

### 6.1 恐慌套利的操作范式 —— Meta 案例复盘

```
事件链:
  T+0  Meta 宣布计划输出自有算力
       → 市场解读: "存储需求下降" → 全球存储板块暴跌 5-8%
       → 情绪: 恐慌 😨😨😨
  
  T+1  多家机构发布研报澄清:
       "Meta自用算力不等于减少外购，存储需求基本面未变"
       → 存储板块止跌企稳
  
  T+2  市场消化澄清信息
       → 存储板块涨停恢复

系统应该做什么:
  T+0 暴跌中:
    1. 新闻语义分析: "Meta输出算力" → 标签: 存储行业/利空/情绪强度 8/10
    2. 关联股票识别: 存储板块所有标的(兆易创新/北京君正/江波龙/...)
    3. 基本面冲击评估: 对目标公司 EPS 的实际影响估算
    4. 恐慌程度量化: 跌幅 vs 基本面实际冲击 → 跌幅 > 合理范围?
    5. 如果跌幅 > 基本面冲击 × 1.5 → 标记为「过度恐慌」
    6. 输出信号: "市场对XX事件反应过度，存储板块存在恐慌抄底机会"
    7. 风险: 等机构澄清后再确认，不做第一个抄底的人
```

### 6.2 情绪信号体系

系统需要实时追踪以下情绪信号，分为三个等级：

**Level 1: 大盘情绪（每日）**

| 信号 | 数据源 | 解读 |
|------|--------|------|
| 涨跌家数比 | AKShare | >3:1 = 狂热，<1:3 = 恐慌 |
| 涨停/跌停家数 | AKShare | 跌停 > 50 家 = 恐慌蔓延 |
| 成交量异常 | AKShare | 放量下跌 = 恐慌抛售，缩量下跌 = 阴跌 |
| 北向资金 | AKShare | 连续净流出/流入的方向和量级 |
| 融资余额变化 | AKShare | 杠杆资金是恐慌/贪婪的放大器 |

**Level 2: 板块情绪（实时）**

| 信号 | 数据源 | 解读 |
|------|--------|------|
| 板块内个股涨跌比 | AKShare | 全板块同跌 = 板块级情绪冲击 |
| 板块资金流向 | AKShare | 主力资金是净流入还是流出 |
| 板块新闻情感分析 | 新闻 API | 突发利空/利好的情感强度和传播速度 |

**Level 3: 事件驱动的恐慌套利信号（实时触发）**

| 信号 | 触发条件 | 套利逻辑 |
|------|---------|---------|
| 📰 突发利空新闻 | 新闻情感 < -0.7 + 关联股票群组下跌 > 3% | 分析：是基本面变化还是情绪冲击？ |
| 📉 过度反应检测 | 跌幅 > 事件基本面冲击估算值 × 1.5 | 标记为「过度恐慌」→ 观察池 |
| 🔄 澄清/反转检测 | 后续出现反向新闻（情感 > +0.3） | 确认过度反应 → 发出抄底信号 |
| 📊 机构观点分歧 | 同一事件，多空观点明显分歧 | 分歧 = 机会，共识 = 危险 |
| 🔢 量价背离 | 大跌但大单资金逆势流入 | 聪明钱在抄底，跟随信号 |

### 6.3 恐慌套利的量化决策树

```
新闻事件触发
    │
    ├─ Step 1: 事件定性
    │   ├─ 是基本面永久性变化？ → 不是套利机会，进入正常分析管道
    │   └─ 是情绪性/暂时性冲击？ → 进入 Step 2
    │
    ├─ Step 2: 冲击范围识别
    │   ├─ 识别直接关联股票（主营相关）
    │   ├─ 识别间接关联股票（供应链/竞品/替代品）
    │   └─ 排除无关股票（被误杀的，反而是最佳抄底目标）
    │
    ├─ Step 3: 基本面冲击量化
    │   ├─ 估算事件对目标公司年化 EPS 的最大影响
    │   ├─ 计算合理跌幅 = EPS影响 × 合理PE
    │   └─ 实际跌幅 vs 合理跌幅
    │
    ├─ Step 4: 过度反应判定
    │   ├─ 实际跌幅 > 合理跌幅 × 1.5 → 过度恐慌 ✅
    │   ├─ 实际跌幅 ≈ 合理跌幅 → 市场有效定价，无套利空间
    │   └─ 实际跌幅 < 合理跌幅 → 市场尚未充分反应，回避
    │
    ├─ Step 5: 抄底时机判断
    │   ├─ 等待止跌信号: 成交量缩小 + 跌幅收窄 + 出现下影线
    │   ├─ 等待澄清信号: 机构研报/公司公告澄清事件影响
    │   └─ 不在第一天下跌时抄底（飞刀还在落）
    │
    └─ Step 6: 仓位与风控
        ├─ 恐慌套利仓位上限: 正常仓位的 50%（套利不是投资）
        ├─ 止损更紧: 亏损 1%（非正常 2%）即平仓
        └─ 持仓周期: 3-5 个交易日（不是长期持有）
```

### 6.4 情绪信号的数据源

| 数据 | 来源 | 频率 |
|------|------|------|
| A股大盘情绪指标（涨跌比/涨停数/成交额） | AKShare | 实时 |
| 板块资金流向 | AKShare | 日 |
| 北向资金 | AKShare / 华泰 skill | 实时 |
| 新闻情感分析 | AKShare 新闻模块 + 关键词触发 | 实时 |
| 机构研报 | 华泰 skill / 国信 skill | 日 |
| 社交媒体情绪（微博/雪球） | last30days-cn skill（免费） | 实时 |
| 公司公告 | AKShare / 国信 skill | 实时 |

### 6.5 恐慌套利不能做的事

```
❌ 不要第一个抄底 — 等止跌信号确认
❌ 不要在暴跌当天全仓抄底 — 分 2-3 批
❌ 不要对所有暴跌都套利 — 只有情绪冲击才适用
❌ 不要重仓套利 — 套利仓位上限是正常仓位的 50%
❌ 不要扛套利变投资 — 3-5 天不涨就撤
❌ 不要对基本面真出问题的票做套利 — 那是接飞刀
```

## 八、5 层路由详细设计（v4 更新：军规层 + 情绪套利）

### 军规层 — 决策纪律门禁（在 L0 之前，纯规则）

详见 [五、投资军规与思维模型](#五投资军规与思维模型--决策的上游过滤器) —— 10 条军规在此处作为硬阻断执行。
军规拒绝的交易，不进入 L0。

### Layer 0 — 保安（纯规则，0% AI）

```
输入: 通过军规层过滤的股票列表
规则:
  1. ST/*ST → REJECT
  2. 上市 < 60 个交易日 → REJECT
  3. 日均成交额 < 5000万 → REJECT
  4. 当日涨跌停 → REJECT（T+1 重新评估）
  5. 停牌中 → REJECT
  6. 近 12 个月有财务造假/违规记录 → FLAG
输出: [ACCEPTED] + [FLAGGED] + [REJECTED]
```

### Layer 1 — 分析师（v8: 四大师方法论 + 量化因子 + 情绪套利）

> v8：借鉴 ai-berkshire 的七步研究流程和四位大师互相挑战机制。
> 四位大师不是分工，而是设计来**互相挑战**——段永平说好生意，芒格会问怎么会死；巴菲特说够便宜，李录会问10年后还在吗。

```
输入: Layer 0 通过的股票列表
前置步骤: 偏见自觉 — 信息丰富度A/B/C评级 + 4项偏见自查

分析维度（七步流程，借鉴 ai-berkshire investment-research）:
  Step 1. 数据收集与校验   → 多源交叉验证，市值/估值手工验算（禁止LLM心算，使用Decimal精确十进制）
  Step 2. 生意本质(段永平)  → 一句话定义生意/收入结构/BMC画布/毛利率/定价权/差异化
  Step 3. 护城河(巴菲特)    → 五类护城河逐一验证(品牌/转换成本/网络效应/规模效应/技术壁垒)
                             + 六关买入Checklist(R>15%+毛利率>40%+正FCF+轻资产+低负债+管理层诚信)
  Step 4. 逆向思考(芒格)     → 列出所有失败路径+历史类比+偏误自查+空方论点收集
  Step 5. 文明趋势(李录)     → 10年后还在吗? 行业范式转移? 永久性资本损失风险?
  Step 6. 估值与安全边际    → 三情景估值(乐观/中性/悲观)+反向DCF（借鉴 ai-berkshire）
  Step 7. 情绪套利扫描      → 恐慌/贪婪极端值检测、过度反应识别

量化因子自动化:
  B1. 价值因子: PE分位/PB分位/股息率/FCF Yield
  B2. 质量因子: ROE连续3年/毛利率稳定性/负债率/经营现金流/净利润
  B3. 动量因子: 20日/60日趋势强度/量价关系/北向资金

AI 介入场景:
  - Step 2-5: LLM生成分析论点（四大师视角）
  - Step 6: LLM辅助三情景假设
  - Step 7: 新闻情感分析 + 事件基本面冲击评估

输出: AnalysisReport {
  dimensions: { business_quality, moat, risk_reverse, trend_certainty, valuation, sentiment },
  master_perspectives: { duan, buffett, munger, li },  // 🆕 四大师各自结论
  consensus_points: [...],      // 🆕 四位大师的共识点
  divergence_points: [...],     // 🆕 四位大师的矛盾点——最宝贵的分析
  bull_case, bear_case,
  sentiment_signal: { level: NORMAL|PANIC|GREED|EXTREME, arb_opportunity: bool }
}
```

### Layer 2 — 法官（加权评分 + 反共识检查）

```
输入: Layer 1 的 AnalysisReport
评分权重:
  基本面 40% + 技术面 20% + 资金面 15% + 宏观适配 15% + 行业景气 10%

反共识检查（借鉴 cyberagent）:
  → 当前价格是否已反映所有已知利好？
  → 市场共识在哪些点上可能错了？
  → 什么观测会证明这个判断错误？（可证伪条件）

输出: Verdict { score: 0-100, confidence: 0.0-1.0, falsifiable_conditions: [...], risks: [...] }
阻断: confidence < 0.6 → 不进入 L3
```

### Layer 3 — 交易员（信号→仓位映射）

```
输入: Layer 2 的 Verdict（confidence ≥ 0.6）
输出信号:
  - score ≥ 75 → 建仓/加仓
  - score 50-74 → 持有/观望
  - score 35-49 → 减仓
  - score < 35  → 清仓/回避

仓位公式:
  base_position = (score - 50) / 50 × macro_cap
  final_position = min(base_position, L4_single_stock_cap, L4_sector_cap)
  macro_cap = 宏观打分卡输出的仓位上限（0-100%）
```

### Layer 4 — 风控官（硬约束，不可覆盖）

```
单票最大仓位:    总资金 20%
单行业最大仓位:  总资金 40%
单笔最大亏损:    本金 2%（触及即平仓）
组合最大回撤:    15%（触及减仓至 50%）
创业板/科创板:   主板仓位上限 × 0.8
黑天鹅熔断:     沪深300 单日 -5% → T+1 禁止开新仓
流动性熔断:     持仓市值 > 日均成交额 5% → 禁止加仓
```

## 七、分层合约（Pydantic 模型）

```python
# 层间数据合约 —— 每层的输入/输出类型

class FactCheckResult(BaseModel):
    status: Literal["VERIFIED", "STALE", "DISPUTED", "MISSING"]
    sources: list[str]
    confidence: float  # 0.0 - 1.0
    checked_at: datetime

class MacroSnapshot(BaseModel):
    timestamp: datetime
    indicators: dict[str, float]  # 10指标值
    composite_score: float        # 加权总分

class SecurityPass(BaseModel):
    symbol: str
    status: Literal["ACCEPTED", "FLAGGED", "REJECTED"]
    flags: list[str]  # 拒绝/标记原因

class AnalysisReport(BaseModel):
    symbol: str
    macro_score: float
    factors: dict[str, float]
    bull_case: str
    bear_case: str
    bottleneck_analysis: str | None  # 物理瓶颈（制造业/科技股）
    upstream_risks: list[str]

class Verdict(BaseModel):
    symbol: str
    score: int           # 0-100
    confidence: float    # 0.0-1.0
    recommendation: Literal["BUY", "ADD", "HOLD", "REDUCE", "SELL"]
    falsifiable: list[str]  # 可证伪条件
    risks: list[str]

class TradeSignal(BaseModel):
    symbol: str
    action: Literal["OPEN", "ADD", "HOLD", "REDUCE", "CLOSE"]
    target_weight: float  # 目标仓位占比
    limits: dict          # L4 施加的限制

class RiskCheck(BaseModel):
    passed: bool
    violations: list[str]
    adjusted_weight: float  # 风控调整后的最终仓位

# ── 用户学习层 ──

class UserLevel(str, Enum):
    BEGINNER = "beginner"
    INTERMEDIATE = "intermediate"
    PRO = "pro"

class DecisionLog(BaseModel):
    """每一笔交易的决策记录"""
    id: str
    timestamp: datetime
    symbol: str
    direction: Literal["BUY", "SELL", "SKIP"]
    system_suggestion: TradeSignal  # 系统的建议
    user_action: TradeSignal        # 用户的实际操作
    user_reason: str                # 用户当时的理由
    outcome_1w: float | None        # 1 周后结果
    outcome_1m: float | None        # 1 月后结果
    lessons: list[str]              # 事后学到的教训

class CalibrationReport(BaseModel):
    """月度置信度校准"""
    period: str  # "2026-06"
    accuracy_by_confidence: dict[str, float]  # {"0.8": 0.76, "0.6-0.8": 0.61}
    system_return: float
    user_return: float
    behavior_gap: float  # 用户行为偏差造成的收益损失

class UserProfile(BaseModel):
    """月度用户能力画像"""
    stock_selection: float    # 选股能力 0-100
    timing: float             # 择时能力 0-100
    risk_discipline: float    # 风控纪律 0-100
    emotion_control: float    # 情绪控制 0-100
    trend: dict[str, str]     # 各项 vs 上月: "↑" "↓" "→"
```
