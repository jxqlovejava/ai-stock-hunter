# 投资备忘录写作工作流

## 概述

高质量的投资备忘录是投资决策的基石。本工作流涵盖从搭建框架到输出 HTML 报告的完整流程。

## 步骤一：确立分析框架

### 1.1 明确分析目标

- 这是一笔什么样的投资？（长期持有 / 事件驱动 / 困境反转）
- 投资期限与预期回报范围？
- 关键假设是什么？

### 1.2 定义备忘录结构

标准章节模板：

1. 投资概要（1 段话概括核心逻辑）
2. 业务理解（商业模式、护城河、竞争格局）
3. 财务分析（收入拆解、利润质量、现金流）
4. 估值分析（DCF + 可比估值）
5. 核心风险（Top 5 风险情景）
6. 监管与 ESG 考量
7. 压力测试与情景分析
8. 交易计划与仓位逻辑

### 1.3 标注信息缺口

- 列出当前无法获取的关键信息
- 标记 [DATA_GAP] 并说明预期获取途径

## 步骤二：系统收集数据

### 2.1 财务数据

- 近 5 年三表（利润表、资产负债表、现金流量表）
- 按业务线/区域拆分的收入构成
- 资本结构变动历史

### 2.2 非财务数据

- 市场份额与渗透率
- 客户集中度（前 5 大客户）
- 供应商集中度
- 研发投入结构

### 2.3 行业竞争格局

- 波特五力分析
- 行业集中度（CR5 / HHI）
- 主要竞争对手概况（规模 / 份额 / 战略）

### 2.4 核查要点

- [ ] 每个数据点标注来源（provider + tier + nature）
- [ ] 推测性数据标记 [SPECULATION]
- [ ] 无法验证的声明标记 [UNSOURCED]
- [ ] 关键数据缺失标记 [DATA_GAP]

## 步骤三：多情景建模

### 3.1 基准情景（概率 50%-60%）

- 延续当前趋势，适度增长
- 当前宏观假设不变
- 最可能的营收与盈利路径

### 3.2 乐观情景（概率 15%-25%）

- 市场份额加速提升
- 宏观/行业顺风
- 利润率超预期扩张

### 3.3 悲观情景（概率 15%-25%）

- 竞争加剧 / 需求萎缩
- 监管打压 / 成本上升
- 多项不利因素叠加

### 3.4 情景输出格式

| 情景 | 概率 | 营收 5 年 CAGR | 终局利润率 | 每股内在价值 |
|------|------|----------------|------------|--------------|
| 乐观 | 20%  | 15%-20%        | 25%        | ￥XXX        |
| 基准 | 55%  | 8%-12%         | 20%        | ￥XXX        |
| 悲观 | 25%  | 0%-5%          | 12%        | ￥XXX        |

概率加权内在价值 = Σ(情景价值 × 概率)

## 步骤四：DCF 锚定估值

1. 执行完整 DCF 估值（参见 dcf-valuation 技能）
2. 三情景各自的 3 因子敏感性矩阵（WACC × g × FCF）
3. 可比公司 PE / PB / PS / EV/EBITDA 交叉验证
4. 输出：估值区间 + 锚定价格 + confidence

## 步骤五：起草备忘录正文

### 章节写作顺序

1. **估值分析**（最依赖数据的部分，优先完成）
2. **财务分析**（估值的数据基础）
3. **业务理解**（定性底座）
4. **核心风险**（反向压力测试成果）
5. **压力测试**（量化所有风险的影响）
6. **交易计划**（综合以上全部分析的输出）
7. **投资概要**（最后写，提炼全文）
8. **监管与 ESG**（标准化检查项）
9. **附录**（数据表、引用、方法论说明）

### 写作原则

- 每个结论都要有数据支撑
- 用表格和图表代替大段文字
- 负面信息与正面信息同等篇幅
- 明确标注所有假设
- 结论不带感情色彩

## 步骤六：自我批判与反向思考

### 6.1 偏差自检

- [ ] 确认性偏差：是否只找了支持结论的证据？
- [ ] 锚定效应：DCF 结果是否受到市场价格的暗示？
- [ ] 后见之明：是否用当前结果解读历史？
- [ ] 过度自信：关键假设是否留有足够 buffer？
- [ ] 叙事谬误：太流畅的故事是否缺少反方证据？

### 6.2 反向压力测试

- 假设"这笔投资一定会亏钱"——列清可能的原因
- 每个原因分配发生的概率
- 检查现有分析是否充分考虑了这些路径

### 6.3 求找论证漏洞

- 派自己充当"做空者"写 3 段反驳
- 检查是否有未考虑的政策/技术/竞争风险
- 确认估值假设偏向保守而非激进

## 步骤七：渲染 HTML

### 7.1 HTML 模板结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>投资备忘录 — {股票名称} ({股票代码})</title>
  <style>
    /* 排版：阅读优先，宽版，衬线+无衬线搭配 */
  </style>
</head>
<body>
  <article>
    <header>...</header>
    <nav id="toc">...</nav>
    <section id="summary">...</section>
    <!-- 各章节 -->
    <section id="appendix">...</section>
    <footer>...</footer>
  </article>
</body>
</html>
```

### 7.2 渲染要求

- 自动生成目录
- 表格带斑马纹与交叉引用
- [DATA_GAP] / [SPECULATION] / [UNSOURCED] 标签用不同颜色高亮
- 估值表使用等宽字体
- 支持打印为 A4 PDF
- 每页页脚标注数据截止日期与作者

## 步骤八：生成研究报告

1. 合并所有章节为完整文档
2. 执行最终一致性检查
3. 输出 HTML + MD 双版本
4. 如配置了发布目录，同步至输出目录

### 最终检查清单

- [ ] 所有 [DATA_GAP] 已声明影响
- [ ] 每个结论有 source_citation
- [ ] confidence 反映数据质量加权
- [ ] 至少两个市场情绪来源交叉验证
- [ ] 自我批判章节已包含
- [ ] 估值区间合理覆盖三情景
- [ ] 交易计划与估值逻辑一致
- [ ] 报告无具体买卖建议（仅分析区间）
