## 🔴 关键缺口（按优先级排序）

### P0: 筹码分布分析 — 完全缺失

**这是 A 股反操纵最重要的能力。** 庄家操纵的前提是筹码集中——如果能识别筹码集中度，就能提前避开操纵高发标的。

**需要的数据（可通过 AKShare/东财获取）：**
- 股东户数及变化趋势（`ak.stock_holder_num()` 或东财 datacenter）
- 前十大股东持股比例及变化
- 前十大流通股东持股集中度
- 人均持股金额
- 户均持股数变化率

**需要实现的指标：**
```
筹码集中度 = f(股东户数变化率, 前十大持股比例, 人均持股, 户均持股变化)
  - 股东户数季度环比下降 > 15% → 高度集中预警
  - 前十大流通股东持股 > 60% → 筹码高度集中
  - 户均持股连续 2 季度上升 > 20% → 吸筹进行中
```

**反操纵意义：**
- 筹码集中的股票 = 庄家控盘能力强 = 操纵风险高
- 不应完全不碰，但必须降低仓位 + 提高操纵检测敏感度
- 筹码分散的股票 = 庄家难以操纵 = 技术分析更有效

### P1: 操纵 Playbook 数据验证 — 有框架但未执行

7 个操纵 playbook 全部是 `evidence_level="HYPOTHESIS"`（逻辑推演，未经数据验证）。PlaybookValidator 只验证了涨停接力/机构抱团/国家队托底，**跳过了所有操纵类 playbook**。

**需要做的事：**
1. 为 7 个操纵 playbook 各实现 `_validate_*` 方法
2. 需要分钟级历史数据来回测模式匹配率（mootdx 可提供历史分钟线）
3. 统计每种模式出现后的次日/3日/5日价格走势
4. 验证"检测到诱多信号→次日下跌"的统计显著性
5. 自动升级 evidence_level: HYPOTHESIS → PRELIMINARY → CONFIRMED → CALIBRATED

**反操纵意义：**
- 知道哪些操纵模式真的有效（高胜率预测），哪些是噪音
- 减少误报（false positive）— 避免把正常波动当操纵
- 量化每种操纵模式的风险溢价

### P2: 多日操纵模式检测 — 仅日内、缺跨日

当前 ManipulationDetector 只分析单日分钟线。很多庄家操纵是跨日运作的：

| 跨日模式 | 周期 | 特征 |
|---------|:--:|------|
| 横盘吸筹 | 2-8 周 | 窄幅震荡+缩量+股东户数下降 |
| 连续拉升出货 | 3-7 天 | 连阳+量递增→突然放量阴线 |
| 涨停板诱多链 | 2-4 天 | 连续涨停→开板放量→次日闷杀 |
| 对倒操纵链 | 5-15 天 | 量价背离+锯齿走势+无基本面支撑 |
| 消息配合操纵 | 1-5 天 | 利好消息+高开低走/利好兑现出货 |

**需要实现：**
- 日线级操纵模式识别器（`DailyManipulationDetector`）
- 横盘吸筹检测：窄幅震荡 + 缩量 + 股东户数下降
- 连阳出货检测：N 连阳 + 量递增 + 突然放量长阴
- 消息-价格背离检测：利好消息 + 高开低走 = 利好出货

### P3: 资金面深度追踪 — 框架空壳

`game_theory/capital_flow.py` 当前只是从 `rules.py` re-export，没有任何实际计算。

**需要的数据（可通过东财 datacenter 获取）：**
- 超大单/大单/中单/小单净流入（`ak.stock_fund_flow_individual()` 或东财 `RPT_MONEYFLOW_DAILY`）
- 主力资金连续流入/流出天数
- 资金流向与价格走势背离度

**需要实现的指标：**
```
资金背离度 = 价格方向 vs 主力资金方向
  - 价格涨 + 主力流出 > 3天 → 诱多出货预警
  - 价格跌 + 主力流入 > 3天 → 诱空吸筹预警
  - 价格涨 + 主力流入 → 真实上涨（低操纵风险）

主力参与度 = 超大单净额 / 总成交额
  - > 15% → 主力高度活跃（可能是操纵也可能是真趋势）
  - < 3%  → 散户市（技术分析更有效）
```

### P4: 反操纵仓位联动 — 检测到但未影响决策

当前操纵检测结果在 `t0_decision.py` 中仅作为加减分项（-10 到 -25 分），没有系统地影响：
- **仓位大小**：高操纵风险 → Kelly 仓位应打折扣
- **止损策略**：不同操纵模式需要不同止损逻辑（诱多止损应更快，洗盘应更宽容）
- **入场延迟**：检测到可疑信号 → 推迟入场 1-2 天观察
- **退出加速**：确认操纵出货 → 立即减仓而非等待技术止损

**需要实现：**
```
操纵调整后仓位 = Kelly仓位 × (1 - 操纵风险评分/100)
  - 操纵风险 0-30 分 → 正常仓位
  - 操纵风险 30-60 分 → 仓位 × 0.7
  - 操纵风险 60-100 分 → 仓位 × 0.3 或不入场

操纵感知止损:
  - 诱多出货 → 跌破当日均价即止损（快）
  - 洗盘震仓 → 放宽止损 2%（避免被洗出）
  - 钓鱼线 → 立即止损（最高风险）
```

### P5: 操纵历史数据库 — 无记忆

系统无法回答"这只股票历史上是否经常被操纵"。

**需要建设：**
- 证监会行政处罚记录（操纵市场罚单）
- 历史异常波动记录（单日涨跌 > 7% 且次日反向 > 5%）
- 历史龙虎榜操纵嫌疑标记
- 高管/大股东减持配合拉升的嫌疑记录
- 个股操纵风险评分（历史累计）

**反操纵意义：**
- "惯犯"股票 — 出现过 3 次以上操纵嫌疑 → 永久标记为高操纵风险
- 首次出现操纵嫌疑 → 可能是假阳性，给更低权重
- 配合大股东减持的操纵 → 最高风险等级

### P6: 情绪-操纵联动 — 两个独立模块未打通

`sentiment/` 模块检测市场情绪（恐慌/贪婪），`game_theory/manipulation/` 检测操纵，但两者没有联动。

庄家操纵高度依赖市场情绪：
- **恐慌中洗盘**：市场情绪 PANIC + 个股急跌 = 大概率是洗盘而非真实崩盘
- **狂热中出货**：市场情绪 GREED + 个股放量滞涨 = 大概率是出货
- **利用消息面**：突发利空 + 个股放量下跌 + V型反弹 = 诱空吸筹

**需要实现：**
```
市场情绪 × 操纵模式 = 操纵置信度调整
  - 市场 PANIC + 检测到诱空吸筹 → 置信度 +0.15（恐慌中洗盘是经典套路）
  - 市场 GREED + 检测到诱多出货 → 置信度 +0.15（狂热中出货是经典套路）
  - 市场 CALM + 检测到操纵 → 置信度不变
```

## 实现路线图建议

### Phase 1 (1-2 周): 数据基础
1. 接入股东户数/前十大股东数据（AKShare）
2. 实现 `ChipConcentrationAnalyzer`（筹码集中度分析）
3. 接入主力资金流向数据（东财 datacenter）
4. 实现 `CapitalFlowAnalyzer`（填补现有空壳）

### Phase 2 (2-3 周): 操纵检测升级
5. 实现 7 个操纵 playbook 的数据验证
6. 建设分钟级历史数据库（mootdx 批量拉取）
7. 实现 `DailyManipulationDetector`（跨日模式）
8. 建设操纵历史数据库

### Phase 3 (1-2 周): 决策联动
9. 操纵感知仓位管理（Kelly × 操纵风险折扣）
10. 操纵感知止损策略
11. 情绪-操纵联动检测
12. 消息-价格背离检测

### Phase 4 (持续): 验证与进化
13. 反操纵回测（with/without 对比）
14. 操纵信号质量追踪（假阳性/假阴性率）
15. Playbook 参数自动校准
16. 个性化操纵风险容忍度
